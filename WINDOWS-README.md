# Windows 一键启动脚本使用说明

## 📁 文件说明

本项目为 Windows 系统提供了统一的一键启动脚本：

- **Windows启动.bat** - Windows 一键启动程序（唯一启动文件）

## 🎯 功能特点

### 统一启动脚本特点

- ✅ **菜单式操作** - 提供可视化菜单，支持启动、停止、查看状态等功能
- ✅ **全中文界面** - 支持中文显示，用户友好
- ✅ **自动环境检查** - 自动检查 Node.js、npm、依赖包
- ✅ **智能端口管理** - 自动检查端口占用并处理冲突
- ✅ **一键启动** - 自动安装依赖、启动服务器、打开浏览器
- ✅ **状态监控** - 实时查看系统运行状态
- ✅ **美观界面** - 使用 emoji 和特殊字符，界面美观
- ✅ **详细帮助** - 内置完整的使用说明和常见问题解答

## 🚀 快速开始

### 启动系统

1. 双击运行 `Windows启动.bat`
2. 选择菜单选项 `[1] 启动系统`
3. 等待系统自动完成环境检查和依赖安装
4. 系统自动打开浏览器，开始使用

### 停止系统

1. 在脚本菜单中选择 `[2] 停止系统`
2. 或直接在服务器窗口按 `Ctrl+C`
3. 或关闭所有相关命令行窗口

## 📋 菜单功能详解

### [1] 🚀 启动系统（推荐）
完整启动流程包括：
- 检查系统文件（index.html、package.json）
- 检查 Node.js 环境（要求 >= 14.0.0）
- 检查 npm 包管理器
- 自动安装依赖包（首次运行）
- 检查端口 3007 占用情况
- 启动 Node.js 服务器
- 自动打开浏览器访问系统

### [2] 🛑 停止系统
- 查找占用端口 3007 的进程
- 安全停止服务器
- 清理相关资源

### [3] 📊 查看运行状态
显示以下信息：
- 服务器运行状态
- 当前端口占用情况
- Node.js 和 npm 版本
- 依赖包安装状态

### [4] 📖 帮助说明
包含：
- 系统功能介绍
- 快速开始指南
- 注意事项
- 常见问题解答
- 技术支持信息

### [5] ❌ 退出
退出程序

## 🔧 系统要求

- **操作系统**: Windows 7/8/10/11
- **Node.js**: >= 14.0.0
- **npm**: >= 6.0.0
- **端口**: 3007（确保未被占用）
- **浏览器**: Chrome、Firefox、Edge 等现代浏览器

## 📍 访问地址

- **服务器地址**: http://localhost:3007
- **前端页面**: http://localhost:3007/index.html
- **API接口**: http://localhost:3007/api

## ⚠️ 常见问题

### Q: 提示"未检测到 Node.js"？
**A:** 请先从 https://nodejs.org/ 下载并安装 Node.js

### Q: 提示"端口 3007 已被占用"？
**A:** 脚本会自动尝试停止占用进程，或选择停止现有服务器

### Q: 浏览器无法访问页面？
**A:**
- 检查防火墙设置
- 确保服务器已启动
- 尝试手动访问：http://localhost:3007

### Q: 依赖安装失败？
**A:**
- 检查网络连接
- 尝试手动运行：`npm install`
- 使用国内镜像：`npm config set registry https://registry.npmmirror.com`

### Q: 如何更新系统？
**A:** 使用 git pull 更新代码后，重新运行启动脚本
